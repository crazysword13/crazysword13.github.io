IDS实验–课下

一、实验内容

借鉴网络扫描实验中对Heartbleed漏洞的利用。同样在kali中使用msf，msf在进行到“exploit“时投放攻击载荷实现攻击。自主编写规则实现针对Heartbleed漏洞攻击的检测。报告中要说明自定义规则的意义。

提示：规则中可以考虑从攻击使用到的服务、端口号、攻击载荷中包含的字段等角度考虑编写规则。

二、实验环境

  • 靶机：Ubuntu + snort + docker
  • 攻击源：kali + msf

三、实验过程

（一）搭建漏洞

1、首先启动docker容器，进入环境所在目录，使用sudo docker-compose up -d启动环境。

​ 可以通过docker ps命令查看已运行环境。

2、打开浏览器，访问链接https://your-ip:8443，出现Heartbleed Test说明环境启动成功。

（二）编写规则实现针对Heartbleed漏洞攻击的检测

alert tcp any any -> $HOME_NET 8443 (msg: "HeartBleed Attack!"; sid:2000005)

规则释义：检测来自8443端口的非法连接请求，从而判断是否有遭遇Heartbleed攻击

之后启动snort，sudo snort -c /etc/snort/snort.conf -A fast -l /var/log/snort -i ens33

（三）在kali中使用msf进行漏洞攻击

接下来使用msf利用漏洞

打开msf后，运用命令search heartbleed查找heartbleed模块。

然后使用use auxiliary/scanner/ssl/openssl_heartbleed 命令选择模块，并用show options指令查看选项

了解选项后设置靶机地址RHOSTS和靶机端口号RPORT。同时设置verbose以看到泄露的信息。

然后输入exploit开始攻击

可以看到，我们已经成功获取了泄露的数据，至此，漏洞利用成功

（四）查看检测效果

在日志中查看检测报告(cat /var/log/snort/alert)：

可以看到，我们成功检测到了Heartbleed的攻击，至此，实验成功。

四、实验总结

在本次实验中，我再一次学习了漏洞的搭建、snort规则的编写以及kali工具的使用等知识，成功搭建了Heartbleed的漏洞环境，并设计了snort规则用于检测Heartbleed的攻击，最后利用kali的工具msfconsle实现了攻击，查看snort日志看到攻击检测结果。通过设计这次的实验，我不仅巩固了已有的知识和技术，还进一步加强了这些知识之间的相关性，取得了不小的收益。
